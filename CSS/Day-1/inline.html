Cloud Computing Overview

Cloud computing is a model for delivering on-demand IT resources (compute, storage, networking, applications, etc.) over the Internet. NIST defines it as “a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources… that can be rapidly provisioned and released with minimal management effort or service provider interaction”. In practice, this means organizations use remote data centers (“the cloud”) owned by providers (CSPs) instead of hosting servers on-site. Companies pay only for the resources they consume (a pay‑as‑you‑go model), avoiding large upfront capital investment. For example, Oracle notes that cloud computing delivers servers, software, storage, databases, development platforms and more over the Internet, with customers accessing needed resources on demand and paying only for what they use

Key Characteristics:

Cloud systems share five essential traits (per NIST). They include on-demand self-service (users can provision resources automatically without human intervention), broad network access (resources available over the network to varied devices), resource pooling (multitenant sharing of provider resources), rapid elasticity (instantaneous scaling up or down), and measured service (resource usage is metered and billed). For instance, providers pool computing capacity to serve multiple tenants, dynamically allocating servers or storage as needed. These characteristics enable flexible, scalable IT: customers spin up servers or storage via a web portal, scale usage automatically during traffic spikes, and pay only for the exact amount of compute/storage

Deployment Models

Public Cloud: Operated by third-party CSPs, public clouds provide shared (multi-tenant) resources over the Internet. Providers like AWS, Azure or Google Cloud offer computing, storage, and software to any customer on a subscription or pay-per-use basis. For example, AWS makes virtual servers (EC2), storage (S3), and managed databases available on demand to multiple tenants. Public clouds offer high scalability and cost-sharing but require trusting the provider with security and compliance.
