Why CallBack ?

A callback is a function passed as an argument to another function. It runs after the first function finishes.

  1. To execute code in a specific order.
  2. Commonly used for asynchronous tasks.

Why Promise?

Callbacks can become deeply nested, leading to Callback Hell (also called the Pyramid of Doom).A Promise provides a cleaner way to handle asynchronous operations.

  1. Avoids callback hell.
  2. Better error handling.
  3. Easier to read.

Why Async/Await?

async/await is built on top of Promises. It makes asynchronous code look like synchronous code.

  1. Cleaner syntax.
  2. Easier to understand.
  3. Better for complex asynchronous code.

Why Fetch?

The Fetch API is used to communicate with servers (APIs).

You can:

  1. Get data (GET)
  2. Send data (POST)
  3. Update data (PUT)
  4. Delete data (DELETE)
